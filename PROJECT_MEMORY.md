## 项目根本定位（必须记住）

> **本项目不是猜比分工具，而是世界杯赔率 / 盘口 / 水位 / 波胆风控分析工具。**

这意味着：

- 页面展示的比分候选，是**基于赔率盘口的风控参考**，不是"必中预测"
- 风控分析关注的是：哪些比赛值得重点跟踪、哪些需要等临场确认、哪些热门可能过热、哪些盘口有防平防冷信号
- 赛后复盘，是为了**验证模型判断是否合理**，不是为了追求命中率数字
- 比分命中 ≠ 方案成功，还要看**覆盖成本、配置比例、回报效率**
- 后续复盘要区分**"分析命中"**（方向、盘口、区间判断对了）和**"方案是否划算"**（赔率配合后是否值得下注）

详见 `PROJECT_GUARDRAILS.md`。

## 复盘规则（最重要的部分，开发前必须读）

### 规则 1：每场比赛分三个版本

每场比赛的数据，在 preview.html 的分析和复盘中，要区分：

- **早盘版**：开赛 24-48 小时前的赔率、盘口、水位、模型判断
- **临盘版**：开赛前 1-6 小时的最新赔率、盘口、水位、模型判断（可能和早盘不同）
- **赛后版**：实际赛果 + 覆盖成本 + 回报效率复盘

**原因**：很多比赛早盘和临盘的赔率、盘口水位会剧烈变化。不看临盘，只看早盘结论，很容易错判模型。

### 规则 2：没有临盘复测，不能简单说"模型错了"

> **即使最终比分和模型推荐不一致，也不能直接说模型错了。**

因为：

- 早盘的判断可能在当时是合理的，但临盘数据变化后应当修正
- 如果你只有早盘数据，赛后复盘只能说"**早盘判断在当时数据下合理，但临盘数据变化未被捕捉**"
- 不能拿着赛果倒过来说"模型应该预测 X-Y"，这是事后诸葛亮

### 规则 3：美国 4-1 巴拉圭的典型教训

这场比赛是一个**方向对、比分强度错**的经典案例：

- ✅ 方向对了：美国胜
- ❌ 比分强度错了：模型可能只预测 1-0 或 2-1，但实际是 4-1（大比分）

**复盘时必须说清楚**：

- 临盘复测（如果有数据的话）最多从 1-0 修正到 2-1 或 3-1
- **绝对不能**看着赛果倒推说"模型应该预测 4-1"——在真实赛前场景下，4-1 这种极端比分不应该是主要推荐
- 要问的问题是：**进球区间判断是否合理**（美国优势是否足够大到进入"3+ 球"区间），**是否有冷门防冷信号**

### 规则 4：比分命中 ≠ 方案成功

> **即使比分正好命中，也不能直接说方案成功。**

必须同时看三个要素：

1. **覆盖成本**：候选比分有几个？每个的赔率是多少？分散下注的成本是否过高？
2. **配置比例**：主方向 / 防平 / 防冷的资金分配是否合理？
3. **回报效率**：投入多少？潜在回报多少？是否值得做？

**反例**：候选了 10 个比分，其中一个正好命中，但赔率很低、其他 9 个都没中——这种"命中"其实不划算。

### 规则 5：复盘要区分"分析命中"和"方案是否划算"

**分析命中**（模型判断层面）：

- 方向是否对（胜/平/负）
- 让球盘口是否合理
- 大小球区间是否对
- 波胆候选是否包含了实际比分
- 是否识别了热门过热/冷门信号

**方案是否划算**（实操层面）：

- 候选比分数量 vs 赔率
- 主方向、防平、防冷的配置比例
- 实际投入 vs 可能回报
- 是否满足最低回报门槛（如至少 1.5 倍以上才值得做）

**一个好的风控判断**：分析层面命中 + 方案层面划算

**一个常见陷阱**：只看比分是否命中，不看方案是否划算

## 数据源可信度分级（dataTier）

```
A 级（官方/权威）：FIFA 官方赛程/比分、官方球队信息、官方比赛状态
B 级（可靠免费 API）：有文档、有稳定接口、有免费额度、允许项目调用、支持 GitHub Actions
C 级（权威媒体/大型体育平台）：公开信息，用于交叉验证阵容/伤停/新闻
D 级（不稳定）：论坛、自媒体、抓取限制不清的网站 → 仅作为人工参考，不自动抓取
```

见 `PENDING_DECISIONS.md` 的 6 个待确认决策。
